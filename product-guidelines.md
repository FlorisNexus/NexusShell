# 🏆 Product & Development Strategy

This master document defines the dual-track strategy for the FlorisNexus ecosystem.

---

## 🏗️ BRANCH 1: LOCAL DIGITAL PRESENCE (FLORISNEXUS)
**Core Mission:** Empower local SMEs and artisans with world-class web technology.
**Business Strategy:** "The Local Hero". High trust, high performance, low friction.

### 🛠️ Technical Standards
- **Stack:** Vite, React/TS, Tailwind CSS, Azure Static Web Apps.
- **Patterns:** SPA (Single Page Application) for "app-like" speed.
- **Performance:** 100/100 Lighthouse scores. Zero layout shift.
- **Automation:** Standardized deployment via GitHub Actions.

### 🎯 Strategic Nuances
- **Conversion First:** Every site must have a clear Call-to-Action (CTA).
- **Mobile-First:** 80%+ of traffic is mobile; the site must be perfect on phone screens.

### 🔍 SEO Optimization (Local Focus)
- **Local SEO:** Deep integration with Google Business Profile.
- **Keywords:** Targeted optimization for specific regions (e.g., Hainaut, Belgium).
- **Schema.org:** Implementation of `LocalBusiness` structured data.
- **Core Web Vitals:** Strict adherence to LCP, FID, and CLS metrics for ranking.

---

## 🌍 BRANCH 2: INTERNATIONAL SAAS (GLOBAL TECH)
**Core Mission:** Build scalable, high-trust technical products for the global market.
**Business Strategy:** "Build in Public". Transparency, technical excellence, and developer-centricity.

### 🛠️ Technical Standards
- **Stack:** .NET 10 (Isolated), Blazor WebAssembly, Azure (Functions, Service Bus).
- **Patterns:** Clean Architecture, Domain-Driven Design (DDD).
- **Data:** Dataverse / SQL Azure. Privacy-first storage.
- **Infrastructure:** Bicep (Infrastructure as Code) for reproducible environments.

### 🎯 Strategic Nuances
- **Founder Transparency:** Share technical challenges, benchmarks, and "Ghost Bugs" publicly.
- **Developer Experience (DX):** API-first design. Documentation is a marketing feature.

### 🔍 SEO Optimization (Global & Technical Focus)
- **Technical SEO:** Automated Sitemaps, Robots.txt, and canonical URLs.
- **Social Metadata:** Rich OpenGraph and Twitter Card integration for "Build in Public" sharing.
- **Content Strategy:** High-intent technical keywords (e.g., ".NET 10", "Azure Functions").
- **Authority Building:** Backlink strategy through Product Hunt, Indie Hackers, and dev communities.

---

## 📜 SHARED WORKSPACE PROTOCOLS
1. **Adaptive Memory:** Every project must maintain a `GEMINI.md` file that evolves with the project.
2. **Git Discipline:** Descriptive, atomic commits. No secrets.
3. **Continuous Delivery:** "If it's in Main, it's in Production."
